## 3. The blocker — pick one

`npm test` baseline fails with `PrismaClientInitializationError: Authentication failed against database server, the provided database credentials for 'radar' are not valid`.

Diagnosis:
- ✅ `DATABASE_URL` (`outreach_db`) works — remote Postgres reachable via SSH tunnel on `127.0.0.1:5432`
- ❌ `DATABASE_URL_TEST` (`radar_test`) fails — the `radar_test` database doesn't exist on the remote server
- Result: 151 of 193 tests fail (all DB-touching tests). Only 42 pure-logic tests pass.

### Option A (recommended) — Create `radar_test` on the remote Postgres server

**Steps (~2 min):**
```sql
-- Connect as the owner of outreach_db via psql or any SQL client
CREATE DATABASE radar_test;
-- Optionally grant the same user that owns outreach_db
GRANT ALL PRIVILEGES ON DATABASE radar_test TO <user>;
```

Then from repo:
```bash
DATABASE_URL="$DATABASE_URL_TEST" npx prisma migrate deploy
npm test  # expect green (or a small set of real failures to fix)
```

**Pros:** Same server, zero infra change, tests isolated from dev data.
**Cons:** Requires ability to `CREATE DATABASE` on the remote — confirm your PG user has this.
**Risk:** None to `outreach_db`. Different database, different name.

### Option B — Spin up a local Docker Postgres for tests only

```bash
docker run -d --name radar-test-pg \
  -e POSTGRES_PASSWORD=radar_test \
  -e POSTGRES_DB=radar_test \
  -p 5433:5432 \
  postgres:16-alpine
```

Update `DATABASE_URL_TEST` in `.env` to point at `postgresql://postgres:radar_test@127.0.0.1:5433/radar_test?schema=public`.

**Pros:** No remote writes. Ephemeral.
**Cons:** Contradicts your "no local postgres" preference. New infra to maintain.

### Option C — Point `DATABASE_URL_TEST` at `outreach_db` (NOT RECOMMENDED)

**Do not do this.** `tests/helpers/testDb.js` calls `TRUNCATE TABLE ... CASCADE` on every `beforeEach` — running tests would wipe your live dev data every single run.

### Option D — Mock the Prisma client in all tests

Rewrite 151 tests to not use real DB. Massive effort. Changes the project's testing philosophy. Not worth it.
